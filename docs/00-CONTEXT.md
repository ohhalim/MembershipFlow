# 00 — 프로젝트 컨텍스트 (새 채팅 인수인계용)

> 이 문서 하나만 읽으면 다른 채팅에서 바로 이어갈 수 있도록 정리함. (2026-06-22 기준)

## 한 줄 요약

골프/콘도 **회원권 시세를 한 곳에 모아 차트로 보고, 목표가 알림 받는** 서비스.
별명: **"회원권판 네이버 증권"**.

## 만드는 사람 / 목표

- 고졸 Java 백엔드 구직자. 현재 서류 통과율 0%, 마구 지원하는 패턴.
- **이 프로젝트의 목표는 "취업 포트폴리오"가 본체, 수익화는 보너스.**
- 기존 포트폴리오 `CoinFlow`(암호화폐 거래소 백엔드)가 있고, MembershipFlow는 그 아키텍처를 재사용한 두 번째 서비스로 만든다.

## 왜 이걸 선택했나 (의사결정 로그)

3주 넘게 아이디어를 탐색함: 골프 매칭 → 테니스 → 명품 관리 → 와인 → 관계/체면 소비 → 회원권 시세.

탈락시킨 이유:
- **골프/테니스 매칭**: 양면시장 → 콜드스타트 지옥. 이미 XGOLF·카카오·골팡 점령. 수수료 받기 어려움.
- **명품 관리 앱**: 수동 입력의 저주(아무도 유지 안 함). 구독만으론 돈 작음.
- **와인/부동산/데이팅**: 주류 통판 규제 / 중개사 라이선스 / 콜드스타트 등 벽.

**회원권 시세 추적을 고른 이유:**
- CoinFlow의 _시세 수집 → 저장 → WebSocket → 차트_ 구조를 **도메인만 바꿔 거의 그대로 재사용** → 면접 스토리 강함.
- **양면시장 아님 / 콜드스타트 없음** (시세는 유저 1명이어도 그 자체로 정보 가치).
- 회원권은 2~3억짜리 자산이면서 주식처럼 시세가 매일 변동 → 추적 수요 명확.
- 현재 거래소 사이트들은 20년 전 UI + 전화 상담 → 깔끔하게 모아 보여주는 것만으로 가치.

## ⚠️ 행동 주의사항 (중요)

본인의 반복 패턴 = **아이디어를 계속 갈아타며 "더 완벽한 것"을 찾는 회피.** 분석이 길어지면 그게 실행을 미루는 회피다.
- 새 아이디어 / 범위 확장 / 수익화 논의로 빠지면 **끊고 구현으로 복귀.**
- 목표는 "더 좋은 아이템"이 아니라 **"끝까지 만든 증거".**
