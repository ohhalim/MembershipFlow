# 05 — 도메인 심화 & 비즈니스 모델

> 2026-06-22 추가 대화에서 정리된 도메인 이해, 경쟁력 분석, 비즈니스 모델, 냉정한 평가.
> (00~04는 "무엇을 만드나", 이 문서는 "왜 경쟁력 있나 / 어떻게 돈 버나 / 한계는 뭔가".)

## 1. 회원권은 어떻게 거래되나 (도메인 이해)

부동산 매매와 유사. 쇼핑몰 결제처럼 한 번에 안 끝남.

거래 흐름:
```
시세 탐색 → 딜러에게 전화 → 매물 수배·가격 절충 → 계약·대금 송금
→ 명의개서(골프장에 서류 제출·명부 갱신) → 세금(취득세·양도세) → 회원증 수령(~1개월)
```

- **모든 거래의 출발점 = "시세 확인 + 매매 타이밍 판단".**
- 명의개서·세무·서류는 딜러가 수수료 받고 **대행**. 부자는 이 귀찮은 행정을 직접 안 함.
- 수수료가 흐르는 자리: 매매가 외 **명의개서료 + 인지세 + 취득세 + 거래소 수수료.**

## 2. 거래 빈도 / 시장 규모 (냉정한 숫자)

| 항목 | 수치 |
|---|---|
| 개인 평균 보유 주기 | 약 3.12년 (5년에 1.5~2회 거래) |
| 주중 회원권(저가형) | 약 1.98년 (갈아타기 잦음) |
| 초고가 회원권 | 약 3.31년 (오래 보유) |
| 연간 전체 거래 건수 | 약 1만 건 내외 |
| 연간 거래 대금 | 약 2.5조 원 (1만 건 × 2.5억) |
| 계절성 | 봄(2~3월)·가을(8~9월) 성수기, 혹한/혹서기 거래 뜸 |

→ **거래 빈도 낮음 + 모수 작음.** 이게 수익화의 핵심 약점.

## 3. 경쟁력 (왜 기존 거래소를 이기나)

각 거래소가 "전국 시세"를 보여줘도 빈틈이 있음:

1. **시세가 파편화됨** — 주식처럼 통일가 아님. 같은 회원권도 거래소마다 가격 다름
   (예: 동아 2.5억 / 에이스 2.48억 / 동부 2.53억). 유저는 3~4개 사이트 일일이 비교해야 함.
2. **개별 거래소는 타사 가격 비교 안 해줌** — 우리는 **독립 애그리게이터**.
   → 스카이스캐너(항공)·호텔스컴바인·다나와 모델.
3. **모바일 UX·목표가 알림 부재** — 기존은 전화 유도용 사이트. Watchlist + 실시간 알림이 없음.
4. **시계열 차트·랭킹·대시보드 부재** — 대부분 엑셀 표/당일 시세. 추이 그래프·상승률 랭킹·자산 대시보드 없음.

> 한 줄: 기존은 "중개 가게", 우리는 "여러 가게 가격을 모아 분석하는 금융 정보 플랫폼".
> 핵심 = 정보의 **비대칭성·파편화** 해결.

## 4. 비즈니스 모델

### 직접 매매결제는 안 함 (포기 결정)
- 명의개서·양도세·인감 등 행정/세무 절차 + 에스크로 + 중개 라이선스 = 1인이 못 넘는 규제 벽.
- → **스카이스캐너식 아웃링크 송객**: 시세 비교 보여주고 "거래 신청/상세"는 실제 거래소로 링크 이동.
  송객 수수료(Referral) 또는 광고비 수취 → 중개 책임에서 자유.

### 수익 구조
| 수익원 | 구조 | 단가 |
|---|---|---|
| 송객 수수료 | 3억 거래의 0.1%만 받아도 건당 30만 원. 월 5건 = 150만 원 | 큼 |
| 개인 프리미엄 구독 | 월 49,000~99,000원 (실시간 최저가 알림 + 주간 리포트) | 중 |
| 법인 구독 | 월 199,000~299,000원 (다중 회원권 평가 대시보드 + 세무 증빙 리포트 + 다중계정) | 중 |

### 가격은 왜 고단가인가
- 타겟이 2~3억 자산 거래자/법인 → **월 9,900원은 잘못된 접근.** 서버비도 안 됨.
- 100~200만 원 싸게 사면 1년 구독료 즉시 회수 → 자산가에게 "의사결정 도구값" 저항 없음.
- 고정비(서버 등) 월 10~20만 원 → **유료 3명이면 BEP, 20명 월 200만, 100명 월 1,000만.**
  소수 핵심 고객만으로 생존하는 니치 고단가 구조.

## 5. 타겟 재정의 (부자는 딜러에게 전화로 끝낸다는 반론 대응)

최상위 부자는 "송도 얼마 되면 연락 주세요" 하고 신경 끔 → 일일이 앱 안 봄.
하지만 **딜러도 사람이라 누락·타이밍 놓침 + 자기 거래소 매물만 알려줌.** 시스템이 이걸 해결.

주 사용자 3부류:
1. **딜러 (B2B)** — 매일 아침 경쟁사 시세표 다 열어 비교. 우리 앱 = 필수 업무 도구. 업무툴엔 돈 안 아낌.
2. **법인 담당자(비서실·총무·재무)** — 상사 보고용 객관적 시세 비교·증빙 리포트 필요. 매일 확인.
3. **3050 젊은 자산가(의사·전문직·IT 창업자)** — 전화 영업 귀찮아함. 앱으로 먼저 확인 후 결정될 때만 전화.

## 6. 냉정한 평가 ("이거 별로 아니냐"의 답)

목표에 따라 갈림:

| 목표 | 평가 |
|---|---|
| 인생역전 창업(유니콘) | ❌ **별로.** 낮은 회전율(3년 1회) + 작은 시장(연 1만 건) = 수수료 천장 낮음. 대박 4/10 |
| 취업 포트폴리오 | ✅ **최상.** 흔한 CRUD와 차별. 도메인 신선 + Batch/스케줄러/시계열/캐싱/이벤트 알림 = 주니어 백엔드 면접 직격 |
| 1인 사이드(월 200~300만) | ✅ **충분히 매력적.** 경쟁자 UX 20년 전 수준 → 유료 100명 확보 현실적 |

> 결론: 본래 목표가 **"취업 포폴 본체 + 출시 보너스"**였으므로 이 아이템은 합리적.
> 거래빈도·시장규모는 *수익* 약점이지 *포폴* 약점이 아님 (면접관은 매출 안 봄).

## 7. 크롤링 입장 (00/02 보강)

- **결정: 지금은 경쟁력 우선으로 실제 시세 크롤링. 출시·수익화 단계에서 리스크 부분만 빼거나 제휴 전환.**
- 정적(Jsoup, robots 허용 사이트 예: 동부) + 동적(Playwright, 막힌 사이트 예: 동아) **둘 다.**
  수집기를 `PriceCollector` 인터페이스로 추상화 → 사이트별 구현체 교체(전략 패턴). 이 자체가 설계 어필.
- 리스크 완화: 단순 복제 ❌ → **집계/평균/지수화로 가공**(내 데이터화), 출처 표기 + 원본 송객(경쟁 아닌 협력), robots 존중, 느린 수집.
- 참고 판례: 잡코리아 vs 사람인, 야놀자 vs 여기어때 (경쟁 서비스의 DB 복제가 쟁점이었음).
- UI 레퍼런스: **토스증권** 따라가기.
