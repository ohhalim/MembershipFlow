# 02 — PRD (제품 요구사항)

## 제품 정의

골프/콘도 회원권 시세를 한 곳에 모아 차트로 보고, 목표가 알림을 받는 서비스.

## 타겟 유저

- 회원권 보유자 / 매수 검토자 (= 골프 치는 자산가).
- "내 회원권 지금 얼마지?", "이거 살 타이밍인가?"를 깔끔하게 확인하고 싶은 사람.

## MVP 범위 (이것만, 더 안 붙임)

1. **회원권 종목 목록 + 검색** — 종목 리스트, 이름/지역/종류 필터.
2. **종목 상세 + 시세 차트** — 현재 시세 + 일별 추이 그래프.
3. **관심 종목 찜 + 목표가 알림** — 찜한 종목이 목표가 도달 시 WebSocket 알림.
4. **시세 자동 수집 배치** — 수집 소스 1~2개부터, N개로 확장 가능한 구조.
5. **회원 / 인증** — 가입/로그인 (CoinFlow 구조 참고).

## MVP에서 제외 (README "확장 계획"에 글로만)

- 거래소 송객 수수료 / 결제 / 프리미엄 구독
- 회원권 가격지수 산출
- 수집 소스 10개+ (설계는 N개 가능하게, 구현은 1~2개)
- 직접 매물 중개/거래

## 준법 정책 (수집)

- robots.txt 존중, 출처 표기, 과도한 요청 금지(rate limit).
- **시세 "정보 집계/표시"만.** 매물 원문 통째 복제 / 직접 중개는 하지 않음.
- 시세 정보 집계는 보통 회색이지만 OK, 매물 DB 복제는 위험.

## 성공 기준 (포트폴리오 관점)

- "시세가 DB에 자동으로 쌓이고, 차트로 보이고, 목표가 알림이 실시간으로 간다"가 동작.
- 면접 어필 포인트: 외부 데이터 수집(배치) + 시계열 저장 + WebSocket 실시간 + 수집 소스 추상화 설계.
